Cloud Run

Cloud Run is a serverless platform, it abstracts away infrastructure so you can focus on your code.

Autoscaling

A Cloud Run service uses autoscaling of container instances to serve requests. This means it starts and stops container instances automatically to handle all requests.

Efficiency

Scaling to zero. When no one is using the service, Cloud Run scales down to zero instances. No compute, no cost.

Concurrency

Concurrency: A single container can handle multiple requests at the same time.

Demo Setting

Single concurrency. In this demo, we limit concurrency to one, which means every container instance can handle only one request at a time.
